Start UI [web] uses Tailwind CSS 4 with CSS variables for a flexible, customizable theming system. The theme supports light and dark modes with automatic OS detection.

## Theme Configuration

The theme is defined in `src/styles/app.css` using CSS variables and Tailwind's `@theme` directive.

## Dark Mode Implementation

Dark mode is implemented using [next-themes](https://github.com/pacocoursey/next-themes) with class-based theme switching.

### Theme Provider

The `ThemeProvider` wraps your application in `src/providers.tsx`:

```tsx
import { ThemeProvider } from 'next-themes';
import type { ReactNode } from 'react';

export const Providers = (props: { children: ReactNode }) => {
  return (
    <ThemeProvider
      attribute="class"
      storageKey="theme"
      disableTransitionOnChange
    >
      {props.children}
    </ThemeProvider>
  );
};
```

<Info>
The `attribute="class"` setting adds a `.dark` class to the root element when dark mode is active.
</Info>

### Theme Switcher Component

The built-in `ThemeSwitcher` component allows users to toggle between light, dark, and system themes:

```tsx
import { ThemeSwitcher } from '@/components/ui/theme-switcher';

export function Header() {
  return (
    <header>
      <ThemeSwitcher />
      {/* Icon only variant */}
      <ThemeSwitcher iconOnly />
    </header>
  );
}
```

### Using Theme in Components

Access the current theme using the `useTheme` hook:

```tsx
import { useTheme } from 'next-themes';
import { MoonIcon, SunIcon, SunMoonIcon } from 'lucide-react';

export function CustomThemeToggle() {
  const { theme, setTheme } = useTheme();

  return (
    <button onClick={() => setTheme(theme === 'dark' ? 'light' : 'dark')}>
      {theme === 'dark' ? <SunIcon /> : <MoonIcon />}
    </button>
  );
}
```

## Custom Dark Mode Variants

Tailwind is configured with a custom dark mode variant in `src/styles/app.css`:

```css
@custom-variant dark (&:is(.dark *));
```

This allows you to use the `dark:` prefix in your components:

```tsx
<div className="bg-white text-black dark:bg-neutral-900 dark:text-white">
  Content that adapts to dark mode
</div>
```

## Typography Scale

Custom text sizes extend Tailwind's defaults:

```css
@theme {
  --text-2xs: 0.625rem; /* 10px */
  --text-3xs: 0.5rem;   /* 8px */
}
```

Use them with Tailwind classes:

```tsx
<p className="text-2xs">Extra small text</p>
<p className="text-3xs">Ultra small text</p>
```

## Custom Breakpoints

Additional responsive breakpoints:

```css
@theme {
  --breakpoint-2xs: 24rem; /* 384px */
  --breakpoint-xs: 32rem;  /* 512px */
}
```

```tsx
<div className="grid grid-cols-1 2xs:grid-cols-2 xs:grid-cols-3 sm:grid-cols-4">
  {/* Responsive grid */}
</div>
```

## Safe Area Insets

For mobile PWA support, safe area insets are available:

```css
@theme {
  --spacing-safe-top: env(safe-area-inset-top);
  --spacing-safe-bottom: env(safe-area-inset-bottom);
  --spacing-safe-left: env(safe-area-inset-left);
  --spacing-safe-right: env(safe-area-inset-right);
}
```

```tsx
<div className="pb-safe-bottom">
  Content with safe area padding
</div>
```

## Customizing Colors

To customize the color scheme:

<Steps>
  <Step title="Update Color Palette">
    Modify the OKLCH values in `src/styles/app.css`:
    ```css
    @theme {
      --color-neutral-900: oklch(0.25 0.01 290); /* Your custom dark color */
    }
    ```
  </Step>
  <Step title="Update Design Tokens">
    Map your colors to semantic tokens:
    ```css
    :root {
      --primary: var(--color-blue-600);
      --primary-foreground: var(--color-white);
    }
    ```
  </Step>
  <Step title="Test Both Themes">
    Verify your changes work in both light and dark modes using the theme switcher.
  </Step>
</Steps>

<Warning>
When using OKLCH colors, ensure you maintain sufficient contrast ratios for accessibility (minimum 4.5:1 for normal text, 3:1 for large text).
</Warning>

## Base Styles

Global base styles are applied in `src/styles/app.css`:

```css
@layer base {
  * {
    @apply border-border outline-ring/50;
  }
  body {
    @apply bg-neutral-50 text-foreground dark:bg-neutral-950;
  }
}
```
