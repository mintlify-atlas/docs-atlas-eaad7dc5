# Quickstart Guide

Get your Start UI [web] application up and running in just a few minutes with our scaffold command.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** ≥ 22 ([Download](https://nodejs.org))
- **pnpm** package manager ([Install pnpm](https://pnpm.io/installation))
- **Docker Desktop** (optional, but recommended for database) ([Download](https://www.docker.com/products/docker-desktop/))

<Tip>
  If you don't have pnpm installed, you can install it globally with:
  ```bash
  npm install -g pnpm
  ```
</Tip>

## Create Your App

<Steps>
  <Step title="Scaffold the project">
    Run the create command to scaffold a new Start UI [web] project:
    
    ```bash
    pnpm create start-ui -t web myApp
    ```
    
    This will create a new folder called `myApp` with the latest version of Start UI [web].
    
    <Info>
      Replace `myApp` with your preferred project name. The command will create a new directory with that name in your current location.
    </Info>
  </Step>

  <Step title="Navigate to your project">
    Change into your newly created project directory:
    
    ```bash
    cd myApp
    ```
  </Step>

  <Step title="Set up environment variables">
    Copy the example environment file to create your local configuration:
    
    ```bash
    cp .env.example .env
    ```
    
    The `.env.example` file contains all necessary environment variables with sensible defaults for local development.
    
    <Warning>
      Make sure to update the `AUTH_SECRET` in your `.env` file. You can generate a secure secret with:
      ```bash
      npx @better-auth/cli@latest secret
      ```
    </Warning>
  </Step>

  <Step title="Install dependencies">
    Install all project dependencies using pnpm:
    
    ```bash
    pnpm install
    ```
    
    This will install all necessary packages and run post-install scripts including Prisma client generation.
  </Step>

  <Step title="Initialize Docker containers">
    Start the required services (PostgreSQL, MinIO, and Maildev) using Docker:
    
    ```bash
    pnpm dk:init
    ```
    
    This command starts:
    - **PostgreSQL** - Database server (port 5432)
    - **MinIO** - S3-compatible file storage (API: port 9000, UI: port 9001)
    - **Maildev** - Email testing server (SMTP: port 1025, UI: port 1080)
    
    <Note>
      Don't have Docker? You can use an existing PostgreSQL database by updating the `DATABASE_URL` in your `.env` file, then skip to step 6 and run `pnpm db:push` and `pnpm db:seed` separately.
    </Note>
  </Step>

  <Step title="Initialize the database">
    Push the Prisma schema to your database and seed it with initial data:
    
    ```bash
    pnpm db:init
    ```
    
    This command:
    - Pushes the database schema to PostgreSQL
    - Seeds the database with example data (users, books, genres, etc.)
  </Step>

  <Step title="Start the development server">
    Launch the development server:
    
    ```bash
    pnpm dev
    ```
    
    Your application will be available at:
    
    ```
    http://localhost:3000
    ```
    
    <Info>
      The development server includes hot module replacement (HMR) for instant feedback as you code.
    </Info>
  </Step>
</Steps>

## Useful Commands

Here are some common commands you'll use during development:

```bash
# Development
pnpm dev                    # Start development server
pnpm build                  # Build for production
pnpm start                  # Start production server

# Database
pnpm db:push                # Push schema changes to database
pnpm db:seed                # Seed database with example data
pnpm db:ui                  # Open Prisma Studio to view/edit data

# Docker
pnpm dk:start               # Start Docker containers
pnpm dk:stop                # Stop Docker containers
pnpm dk:clear               # Remove containers and volumes

# Testing
pnpm test                   # Run unit tests
pnpm test:ui                # Run tests with UI
pnpm e2e                    # Run E2E tests
pnpm e2e:ui                 # Run E2E tests with Playwright UI

# Code Quality
pnpm lint                   # Run ESLint and TypeScript checks
pnpm pretty                 # Format code with Prettier

# Storybook
pnpm storybook              # Start Storybook dev server
pnpm storybook:build        # Build Storybook for production
```
