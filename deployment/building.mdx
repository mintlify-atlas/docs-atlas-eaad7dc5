Start UI [web] uses Vite and TanStack Start to build optimized production bundles with server-side rendering capabilities.

## Build Process Overview

The build process compiles your TypeScript code, optimizes assets, and generates both client and server bundles using Nitro.

### Vite Configuration

The project uses a custom Vite configuration located in `vite.config.ts:14-46`:

```typescript vite.config.ts
export default defineConfig(({ mode }) => {
  // Load env file based on `mode` in the current working directory.
  const env = loadEnv(mode, process.cwd(), 'VITE_');
  return {
    server: {
      port: env.VITE_PORT ? Number(env.VITE_PORT) : 3000,
      strictPort: true,
    },
    plugins: [
      devtools(),
      tsConfigPaths(),
      tanstackStart(),
      nitro({
        modules: [
          (nitro) => {
            nitro.hooks.hook('build:before', () => {
              nitroRetrieveServerDirHook(nitro);
            });
          },
        ],
        routeRules: { '/storybook': { redirect: '/storybook/' } },
      }),
      // react's vite plugin must come after start's vite plugin
      viteReact({
        babel: {
          plugins: ['babel-plugin-react-compiler'],
        },
      }),
      // Copy prisma binaries at the end
      prismaCopyBinariesPlugin(),
    ],
  };
});
```

## Production Build Commands

<Steps>

<Step title="Install dependencies">
Install all production and development dependencies:

```bash
pnpm install
```
</Step>

<Step title="Generate build info (automatic)">
Build information is automatically generated during the build process. This creates a build metadata file that includes:

- Git commit hash (short and full)
- Build date and time
- Version display string

This is handled by `src/features/build-info/script-to-generate-json.ts:39-47`:

```typescript
const generateAppBuild = () => {
  try {
    fs.writeFileSync(generatedPath, JSON.stringify(getContent(), null, 2));
    console.log(`✅ Build info file generated (${generatedPath})`);
  } catch (error) {
    console.error(error);
    throw new Error(`❌ Failed to generate build info file (${generatedPath})`);
  }
};
```
</Step>

<Step title="Generate Prisma client">
Generate the Prisma client before building:

```bash
pnpm gen:prisma
```

This generates the Prisma client in `src/server/db/generated` as configured in `prisma/schema.prisma:4-8`:

```prisma
generator client {
  provider     = "prisma-client"
  output       = "../src/server/db/generated"
  moduleFormat = "esm"
}
```
</Step>

<Step title="Build the application">
Run the production build command:

```bash
pnpm build
```

This command from `package.json:15`:

```json
"build": "cross-env NODE_ENV=production run-p env gen:build-info && vite build"
```

The build process:
1. Sets `NODE_ENV` to `production`
2. Validates environment variables (client and server)
3. Generates build information
4. Runs Vite build with TanStack Start plugin
5. Compiles TypeScript to JavaScript
6. Bundles and optimizes client assets
7. Generates server bundle with Nitro
8. Copies Prisma binaries to output directory
</Step>

</Steps>

## Build Output

After running the build command, the following directory structure is created:

```
.output/
├── server/
│   ├── index.mjs          # Server entry point
│   └── ...                # Server bundles and assets
└── public/
    └── ...                # Static client assets
```

<Info>
The `.output` directory is generated by Nitro and contains everything needed to run your application in production.
</Info>

## Build Optimizations

### TypeScript Configuration

The project uses strict TypeScript settings for better type safety (`tsconfig.json:2-23`):

```json
{
  "compilerOptions": {
    "jsx": "react-jsx",
    "moduleResolution": "Bundler",
    "module": "ESNext",
    "target": "ES2022",
    "skipLibCheck": true,
    "strictNullChecks": true,
    "noUncheckedIndexedAccess": true,
    "noImplicitAny": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

### React Compiler

The build uses the React Compiler for automatic optimization (`vite.config.ts:37-40`):

```typescript
viteReact({
  babel: {
    plugins: ['babel-plugin-react-compiler'],
  },
})
```

### Prisma Binaries

Prisma binaries are automatically copied to the server output directory during the build process (`vite.config.ts:48-64`):

```typescript
function createPrismaCopyBinariesPlugin() {
  let serverDir = '';
  return {
    nitroRetrieveServerDirHook: (nitro: Nitro) => {
      serverDir = nitro.options.output.serverDir.replace(resolve('.'), '.');
    },
    prismaCopyBinariesPlugin: () => ({
      name: 'prisma-copy-binaries',
      writeBundle: async (outputOptions: { dir?: string }) => {
        const outputDir = outputOptions.dir?.replace(resolve('.'), '.');
        if (outputDir === serverDir) {
          await cpy('./src/server/db/generated/**/*.node', resolve(serverDir));
        }
      },
    }),
  };
}
```

<Warning>
Ensure Prisma binaries are included in your deployment package, especially when deploying to different architectures (e.g., ARM vs x64).
</Warning>

## Building Storybook (Optional)

If you want to deploy the component library documentation alongside your application:

```bash
pnpm storybook:build
```

This builds Storybook and moves it to `public/storybook`, making it accessible at `/storybook` in production (`package.json:21`):

```json
"storybook:build": "storybook build && mv ./storybook-static ./public/storybook"
```

<Note>
The Storybook build is optional. Only include it if you want to expose your component documentation in production.
</Note>

## Environment Variables

Environment variables are validated at build time. Make sure all required variables are set:

- **Client variables**: Prefixed with `VITE_` (validated by `src/env/client.ts`)
- **Server variables**: All other environment variables (validated by `src/env/server.ts`)

<Tip>
The build will fail if required environment variables are missing, preventing deployment with invalid configuration.
</Tip>

## Verifying the Build

After building, verify the output:

```bash
# Check if build artifacts exist
ls -la .output/server/index.mjs
ls -la .output/public/

# Check build size
du -sh .output/
```
