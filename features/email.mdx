Start UI [web] uses [React Email](https://react.email/) for email templates and [Nodemailer](https://nodemailer.com/) for sending emails.

## Overview

The email system includes:

- **React Email**: Type-safe email templates with React components
- **Nodemailer**: Reliable email delivery
- **MailDev**: Email testing in development
- **i18n Support**: Internationalized email content
- **Preview Route**: Preview emails in the browser during development

## Configuration

### Environment Variables

Configure email settings in `.env`:

```bash .env
# SMTP Configuration
EMAIL_SERVER="smtp://0.0.0.0:1025"
EMAIL_FROM="Start UI <noreply@example.com>"

# MailDev (Development)
DOCKER_MAILDEV_SMTP_PORT="1025"
DOCKER_MAILDEV_UI_PORT="1080"
```

### Email Transport

The Nodemailer transport is configured in `src/server/email.tsx`:

```tsx src/server/email.tsx
import { render } from '@react-email/render';
import nodemailer from 'nodemailer';
import { ReactElement } from 'react';

const transport = nodemailer.createTransport(envServer.EMAIL_SERVER);

export const sendEmail = async ({
  template,
  ...options
}: Omit<MailOptions, 'html'> &
  Required<Pick<MailOptions, 'subject'>> & { template: ReactElement }) => {
  if (envClient.VITE_IS_DEMO) {
    return;
  }

  const html = await render(template);
  return transport.sendMail({
    from: envServer.EMAIL_FROM,
    html,
    ...options,
  });
};
```

## Email Templates

Create email templates using React components.

### Template Structure

```tsx src/emails/templates/login-code.tsx
import { Container, Heading, Section, Text } from '@react-email/components';
import i18n from '@/lib/i18n';
import { EmailFooter } from '@/emails/components/email-footer';
import { EmailLayout } from '@/emails/components/email-layout';
import { styles } from '@/emails/styles';

export const TemplateLoginCode = (props: {
  language: string;
  code: string;
}) => {
  i18n.changeLanguage(props.language);
  
  return (
    <EmailLayout
      preview={i18n.t('emails:loginCode.preview')}
      language={props.language}
    >
      <Container style={styles.container}>
        <Heading style={styles.h1}>
          {i18n.t('emails:loginCode.title')}
        </Heading>
        
        <Section style={styles.section}>
          <Text style={styles.text}>
            {i18n.t('emails:loginCode.intro')}
          </Text>
          
          <Text style={styles.code}>{props.code}</Text>
          
          <Text style={styles.textMuted}>
            {i18n.t('emails:loginCode.validityTime', {
              expiration: AUTH_EMAIL_OTP_EXPIRATION_IN_MINUTES,
            })}
            <br />
            {i18n.t('emails:loginCode.ignoreHelper')}
          </Text>
        </Section>
        
        <EmailFooter />
      </Container>
    </EmailLayout>
  );
};

export default TemplateLoginCode;
```

## Sending Emails

### Basic Usage

```tsx
import { sendEmail } from '@/server/email';
import TemplateLoginCode from '@/emails/templates/login-code';

await sendEmail({
  to: 'user@example.com',
  subject: 'Your login code',
  template: <TemplateLoginCode language="en" code="123456" />,
});
```

### With Better Auth

Integration with Better Auth email OTP:

```tsx src/server/auth.tsx
import i18n from '@/lib/i18n';
import { sendEmail } from '@/server/email';
import TemplateLoginCode from '@/emails/templates/login-code';
import { getUserLanguage } from '@/server/utils';

emailOTP({
  async sendVerificationOTP({ email, otp, type }) {
    await match(type)
      .with('sign-in', async () => {
        await sendEmail({
          to: email,
          subject: i18n.t('emails:loginCode.subject', {
            lng: getUserLanguage(),
          }),
          template: (
            <TemplateLoginCode
              language={getUserLanguage()}
              code={otp}
            />
          ),
        });
      })
      .with('email-verification', async () => {
        // Implement email verification template
      })
      .exhaustive();
  },
})
```

### Multiple Recipients

```tsx
await sendEmail({
  to: ['user1@example.com', 'user2@example.com'],
  subject: 'Notification',
  template: <NotificationTemplate />,
});
```

### With Attachments

## Development Preview

Preview emails in the browser during development.

### Preview Route

```ts src/routes/api/dev.email.$template.ts
import { previewEmailRoute } from '@/server/email';

export const Route = createFileRoute('/api/dev/email/$template')({
  server: {
    handlers: {
      GET: async ({ params, request }) => {
        const url = new URL(request.url);
        const props = Object.fromEntries(url.searchParams);
        
        return previewEmailRoute(params.template, props);
      },
    },
  },
});
```

### Preview Function

```tsx src/server/email.tsx
export const previewEmailRoute = async (
  template: string,
  props: Record<string, string>
) => {
  let Email;
  try {
    const EmailModule = await import(`../emails/templates/${template}.tsx`);
    Email = EmailModule.default;
  } catch {
    return new Response('Template not found', { status: 404 });
  }

  const html = await render(
    <Email language={props?.language ?? DEFAULT_LANGUAGE_KEY} {...props} />
  );

  return new Response(html, {
    status: 200,
    headers: { 'Content-Type': 'text/html' },
  });
};
```

### Preview URL

Access email previews in your browser:

```
http://localhost:3000/api/dev/email/login-code?code=123456&language=en
```

## MailDev (Development)

MailDev catches all emails sent during development.

<Steps>
  <Step title="Start MailDev">
    MailDev starts automatically with Docker:

    ```bash
    docker-compose up -d
    ```
  </Step>

  <Step title="Access MailDev UI">
    Open your browser to: `http://localhost:1080`
  </Step>

  <Step title="View Sent Emails">
    All emails sent by your app appear in the MailDev inbox.
  </Step>
</Steps>

</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Emails not appearing in MailDev">
    Check:
    1. MailDev is running: `docker ps`
    2. SMTP port matches `.env`: `EMAIL_SERVER="smtp://0.0.0.0:1025"`
    3. MailDev UI is accessible: `http://localhost:1080`
  </Accordion>

  <Accordion title="Template not found">
    Ensure:
    1. Template file exists in `src/emails/templates/`
    2. File exports a default component
    3. File name matches preview URL parameter
  </Accordion>

  <Accordion title="Styles not rendering">
    Email clients have limited CSS support:
    - Use inline styles instead of classes
    - Use tables for layout
    - Test in multiple email clients
  </Accordion>

  <Accordion title="Production emails not sending">
    Verify:
    1. SMTP credentials are correct
    2. SMTP server allows your IP
    3. `EMAIL_FROM` domain is verified
    4. Check spam folder
  </Accordion>
</AccordionGroup>
