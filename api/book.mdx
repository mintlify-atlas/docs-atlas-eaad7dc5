## Overview

The Book API provides comprehensive endpoints for managing books in your application. All endpoints require authentication and specific permissions.

## Permissions

Book API endpoints require the following permissions:

- `book:read` - View books
- `book:create` - Create new books
- `book:update` - Update existing books
- `book:delete` - Delete books

## Endpoints

### List All Books

Retrieve a paginated list of books with optional search.

```typescript
GET /books
```

**Source**: `src/server/routers/book.ts:11-84`

#### Authentication

<ParamField path="permissions" type="object">
  Requires `book:read` permission
</ParamField>

#### Query Parameters

<ParamField query="cursor" type="string" optional>
  Cursor for pagination (ID of the last book from the previous page)
</ParamField>

<ParamField query="limit" type="number" default="20">
  Number of books per page (min: 1, max: 100)
</ParamField>

<ParamField query="searchTerm" type="string" default="" optional>
  Search term to filter books by title or author (case-insensitive)
</ParamField>

#### Response

<ResponseField name="items" type="Book[]">
  Array of book objects for the current page
</ResponseField>

<ResponseField name="nextCursor" type="string" optional>
  Cursor for the next page (undefined if no more pages)
</ResponseField>

<ResponseField name="total" type="number">
  Total count of books matching the search criteria
</ResponseField>

#### Book Object

<Expandable title="Book Object Fields">
  <ResponseField name="id" type="string">
    Unique book identifier
  </ResponseField>

  <ResponseField name="title" type="string">
    Book title
  </ResponseField>

  <ResponseField name="author" type="string">
    Book author
  </ResponseField>

  <ResponseField name="genre" type="Genre | null">
    Associated genre object (includes id, name, color)
  </ResponseField>

  <ResponseField name="publisher" type="string | null">
    Publisher name
  </ResponseField>

  <ResponseField name="coverId" type="string | null">
    ID reference to the book cover image
  </ResponseField>

  <ResponseField name="createdAt" type="Date">
    Creation timestamp
  </ResponseField>

  <ResponseField name="updatedAt" type="Date">
    Last update timestamp
  </ResponseField>
</Expandable>

#### Code Example

<CodeGroup>

```typescript React Query
import { orpc } from '@/lib/orpc/client';

function BookList() {
  const { data, isLoading, fetchNextPage, hasNextPage } = 
    orpc.book.getAll.useInfiniteQuery({
      input: (page) => ({
        searchTerm: 'typescript',
        limit: 20,
        cursor: page?.nextCursor,
      }),
      getNextPageParam: (lastPage) => 
        lastPage.nextCursor ? { nextCursor: lastPage.nextCursor } : undefined,
    });

  if (isLoading) return <div>Loading...</div>;

  return (
    <div>
      {data?.pages.flatMap(page => page.items).map(book => (
        <div key={book.id}>
          <h3>{book.title}</h3>
          <p>by {book.author}</p>
          {book.genre && <span>{book.genre.name}</span>}
        </div>
      ))}
      
      {hasNextPage && (
        <button onClick={() => fetchNextPage()}>
          Load More
        </button>
      )}
      
      <p>Total: {data?.pages[0]?.total} books</p>
    </div>
  );
}
```

```typescript REST API
const params = new URLSearchParams({
  searchTerm: 'typescript',
  limit: '20',
});

const response = await fetch(`/api/rest/books?${params}`, {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
  },
});

const data = await response.json();
// data.items: Book[]
// data.nextCursor: string | undefined
// data.total: number
```

```bash cURL
curl "https://your-app.com/api/rest/books?searchTerm=typescript&limit=20" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

<CodeGroup>

```typescript React Query
import { orpc } from '@/lib/orpc/client';
import { useParams } from '@tanstack/react-router';

function BookDetail() {
  const { bookId } = useParams({ from: '/books/$bookId' });
  
  const { data: book, isLoading, error } = orpc.book.getById.useQuery({
    input: { id: bookId },
  });

  if (isLoading) return <div>Loading book...</div>;
  if (error) return <div>Book not found</div>;

  return (
    <div>
      <h1>{book.title}</h1>
      <p>Author: {book.author}</p>
      <p>Publisher: {book.publisher}</p>
      {book.genre && (
        <span style={{ color: book.genre.color }}>
          {book.genre.name}
        </span>
      )}
    </div>
  );
}
```

```typescript REST API
const response = await fetch('/api/rest/books/book-123', {
  headers: {

  },
});

if (!response.ok) {
  if (response.status === 404) {
    console.error('Book not found');
  }
  throw new Error('Failed to fetch book');
}

const book = await response.json();
```

```bash cURL
curl https://your-app.com/api/rest/books/book-123 \
```

```typescript React Query
import { orpc } from '@/lib/orpc/client';
import { useForm } from 'react-hook-form';

function CreateBookForm() {

      <input {...form.register('author')} placeholder="Author" required />
      <select {...form.register('genreId')} required>

      <input {...form.register('publisher')} placeholder="Publisher" />
      <button type="submit" disabled={isPending}>
        {isPending ? 'Creating...' : 'Create Book'}
      </button>

  );
}
```

```typescript React Query
function EditBookForm({ bookId, initialData }) {
```

```typescript
import { z } from 'zod';
import { zu } from '@/lib/zod/zod-utils';
import { zGenre } from '@/features/genre/schema';

export const zBook = () =>
  z.object({
    id: z.string(),
    title: zu.fieldText.required({ error: t('book:common.title.required') }),
    author: zu.fieldText.required(),
    genre: zGenre().nullish(),
    publisher: zu.fieldText.nullish(),
    createdAt: z.date(),
    updatedAt: z.date(),
    coverId: z.string().nullish(),
  });
```

## Error Handling

<ResponseField name="UNAUTHORIZED" type="401">
  Missing or invalid authentication credentials
</ResponseField>

<ResponseField name="FORBIDDEN" type="403">
  User lacks the required permission (e.g., `book:create`)
</ResponseField>

<ResponseField name="NOT_FOUND" type="404">
  Book with the specified ID does not exist
</ResponseField>

<ResponseField name="CONFLICT" type="409">
  Unique constraint violation (unlikely for books)
</ResponseField>

<ResponseField name="BAD_REQUEST" type="400">
  - Invalid input parameters
  - Foreign key constraint violation (invalid `genreId`)
  - Validation errors
</ResponseField>

### Error Examples

## Search Implementation

The search functionality filters books by title or author (case-insensitive):

```typescript
const where = {
  OR: [
    {
      title: {
        contains: input.searchTerm,
        mode: 'insensitive',
      },
    },
    {
      author: {
        contains: input.searchTerm,
        mode: 'insensitive',
      },
    },
  ],
} satisfies Prisma.BookWhereInput;
```

Search examples:
- `searchTerm: 'typescript'` - Finds books with "typescript" in title or author
- `searchTerm: 'josh'` - Finds books by authors named Josh
- `searchTerm: ''` - Returns all books (no filter)
